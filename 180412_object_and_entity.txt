object 와 entity의 차이

- object
	- '객체 지향 프로그램', 프로그래밍의 패러다임 중 하나로 과거에는 '절차 지향' 등의 다른
	프로그래밍 패러다임이 존재 했었음. 이 때, 객체란, 프로그램을 구성하는 하나의 단위로 
	변수, 메소드, 데이터 값 등이 모두 객체(혹은 개체)로 사용될 수 있음.
	- 객체와 개체를 구분할 경우, 데이터와 그 데이터에 관련되는 동작을 모두 포함한 개념.
	- ex) 기차역에서 승차권을 발매할 경우, entity는 '손님', 동작인 '승차권 주문'은 하나의 객체.
	- object에서 '속성'은 자기 자신의 독특한 특성. 속성을 바꾸면 객체의 특성이 변하게 됨.
	- object에서 method는 객체가 행할 수 있는 것들
	- object에서 event는 객체가 감지할 수 있는 행동

- entity
	- RDBMS에서 entiy는 표현하려는 유형, 무형의 실체로써 구별되는 것을 의미
	- 하나의 개체는 하나 이상의 속성(attribute)으로 구성됨. 속성은 각 개체의 특성이나 상태를 설명
	ex) entity 지주영 -> 속성 : 주민번호, 속성값 : 880000 - 10000000
	- 같은 속성을 가지는 entity 들의 집합은 entity type이라고 함. 
	- 속성(attribute)는 데이터의 가장 작은 논리적인 단위. ex)학번, 이름, 학년, 전공
